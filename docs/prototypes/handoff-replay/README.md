# Handoff Replay Prototype

Throwaway prototype for the deterministic handoff model.

Question:

Can a tiny local UI make the Source/Sink, Flow progress, Surface claim update, and Console operating-state model understandable before implementing server/database work?

Run:

```sh
npm run prototype:handoff
```

Open the printed local URL. The prototype uses in-memory fixture records and does not execute actions, mutate product state, call a server API, or persist data.

Delete or absorb this prototype after the model is validated.
